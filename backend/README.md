# ⚙️ Voxentia Backend

This directory contains the main API Gateway and the core logic for the **Voxentia AI Digital Assistant**.

## 🚀 Technology Stack
- **Framework**: [FastAPI](https://fastapi.tiangolo.com/) (Asynchronous Python)
- **LLM Engine**: [Ollama](https://ollama.com/) (local inference)
- **Vector Storage**: [ChromaDB](https://www.trychroma.com/) (for RAG / Document Analysis)
- **Database**: SQLite / SQLAlchemy (for chat history persistence)
- **Networking**: HTTPX (async requests to TTS/Whisper/Ollama microservices)

## 📂 Directory Structure

```
backend/
├── api/                    # REST API Layer
│   ├── chat.py             # Main streaming chat & multimodal logic
│   ├── documents.py        # PDF processing & ChromaDB management
│   └── transcribe.py       # Proxy for local faster-whisper STT
│
├── services/               # Core AI & Business Logic
│   ├── llm.py              # Ollama client, Tool-Use Loop & Stop-Token management
│   ├── tools.py            # Definition of Python tools (Web Search, Weather, Time)
│   ├── tts.py              # Text-to-Speech synthesis client
│   └── rag.py              # ChromaDB document indexing & retrieval
│
└── core/                   # Infrastructure & Settings
    ├── config.py           # Environment variables & Multi-lingual Persona Prompts
    └── database.py         # Database initialization & models
```

## 🛠️ Key Concepts

### 1. Robust Tool-Use Loop
The `llm.py` service implements a pattern where the model can request tool execution via `[TOOL: name(args)]`. The backend executes the Python function and provides the result as a system message, ensuring a seamless integration of real-time data into conversations.

### 2. Multi-stage Prompt Guarding
To prevent prompt leakage (especially with smaller models like Phi-3), the backend uses a sophisticated set of `stop_tokens` (e.g., `MISSION:`, `RULES:`) and structured system instruction blocks.

### 3. Multimodal Vision Support
The `api/chat.py` endpoint supports base64-encoded images. If an image is provided, it is first analyzed by a vision-capable model (like `llava`) to provide a visual context description to the primary chat LLM.

### 4. Sentence-based TTS Streaming
The backend facilitates a "streaming audio" feel by allowing the frontend to trigger TTS generation for completed sentences while the rest of the text is still being generated by the LLM.

## 🖥️ Local Development (without Docker)

1. **Setup Venv**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # or venv\Scripts\activate on Windows
   ```
2. **Install Deps**:
   ```bash
   pip install -r requirements.txt
   ```
3. **Configure**: Create a `.env` file pointing to your local Ollama/TTS/Whisper instances.
4. **Run**:
   ```bash
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```
